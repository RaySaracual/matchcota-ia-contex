## Branching

- main
- feature/*
- bugfix/*

---

## Commit Rules

- Use meaningful commits.
- Keep commits focused.
- Avoid mixing unrelated changes.
- Reference spec ID or backlog ID in commit message when possible.

---

## Pull Requests

- Require review.
- Require validation.
- Require passing tests.
- Require traceability to acceptance criteria in ai-workspace/specs/.
